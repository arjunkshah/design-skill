# Section Programming

Use this file when the page needs richer section variety.

The goal is to stop the page from becoming one repeated section template with different copy.
This file should be used as a page programming system, not as loose inspiration.

## Rule

Every serious landing page or showcase should be programmed as a sequence.

Do not jump from hero to random sections. Decide the page recipe first, then assign a section form to each slot.

Do not make every section:

- centered
- card-based
- equally dense
- equally animated

## Required Page Recipe

For substantial landing pages, marketing pages, or showcases, define these six slots before generating:

1. `HERO_THESIS`
2. `PROOF`
3. `PRODUCT_STAGE`
4. `DEEP_EXPLANATION`
5. `SOCIAL_PROOF`
6. `CONVERSION`

You may add supporting slots such as:

- comparison
- FAQ
- editorial interlude
- founder note
- pricing
- process walkthrough

But the core six should be accounted for unless the brief clearly does not need one.

## Core Section Types

Choose from these deliberately.

### 1. Central Thesis Section

Use for:

- hero statements
- manifesto moments
- major category breaks

Characteristics:

- one dominant message
- minimal competing elements
- can be static or lightly animated

### 2. Split Section

Use for:

- hero plus proof
- copy plus product frame
- narrative plus visual evidence

Characteristics:

- clear left/right or top/bottom tension
- one side leads, the other supports

### 3. Static Bento Section

Use for:

- feature range
- modular product storytelling
- mixed proof blocks

Characteristics:

- one large anchor block
- smaller surrounding cards
- no need for all cards to animate heavily

### 4. Animated Bento Section

Use for:

- interactive showcases
- launch pages
- products with multiple active states

Characteristics:

- a bento structure with selective motion inside key tiles
- one or two cards feel alive
- the whole grid does not need to move

### 5. Proof Rail Section

Use for:

- trust moments
- enterprise credibility
- dense supporting evidence

Characteristics:

- metrics, logos, quotes, labels, or chips in a side rail or strip
- subordinate to the main thesis

### 6. Framed Product Stage

Use for:

- UI-centric products
- tools with demos or screenshots
- moments where the interface is the persuasion

Characteristics:

- product frame as the center of gravity
- support text or notes around it

### 7. Sticky Story Section

Use for:

- workflows
- comparisons
- product explanation

Characteristics:

- one preview stays pinned
- adjacent text advances
- the content changes while the shell remains stable

### 8. Horizontal Rail Section

Use for:

- gallery sequences
- logo or proof bands
- multiple related modules that benefit from lateral movement

Characteristics:

- horizontal movement inside a bounded area
- strong visual grouping
- should still degrade gracefully on mobile

### 9. Scroll Stack Section

Use for:

- layered cards
- narrative launches
- step accumulation

Characteristics:

- sections or cards build depth as the user continues downward
- overlap is controlled, not chaotic

### 10. Static Reset Section

Use for:

- calming the page after motion-heavy sequences
- restoring legibility
- introducing a new chapter

Characteristics:

- low motion
- strong whitespace
- cleaner, flatter composition

## Programming Rules

- do not use the same section form for more than two adjacent slots
- if one section is highly animated, the next section should usually decompress
- proof must appear early enough to support the claim, not only near the footer
- product-stage sections should feel like persuasion, not like a random screenshot dump
- conversion should feel earned by the narrative, not bolted on at the end

## Good Mixes

Examples:

- Hero Thesis as split + Proof as proof rail + Product Stage as pinned demo + Deep Explanation as sticky story + Social Proof as comparison band + Conversion as quiet reset
- Hero Thesis as centered + Proof as bento + Product Stage as framed product stage + Deep Explanation as split + Social Proof as proof rail + Conversion as centered
- Hero Thesis as editorial quote + Proof as split + Product Stage as horizontal rail + Deep Explanation as scroll stack + Social Proof as bento + Conversion as split

## Bad Mixes

Reject pages that feel like:

- five stacked centered sections
- a hero followed by identical card rows
- motion in every section with no quiet contrast
- every section trying to be the hero
- every slot solved with the same bento shell
- no clear difference between proof, explanation, and conversion

## Section Contrast

Create contrast through:

- static vs animated
- dense vs airy
- centered vs asymmetric
- framed vs open
- vertical flow vs horizontal bias

That contrast is what makes the page feel intentional.
