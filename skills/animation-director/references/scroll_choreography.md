# Scroll Choreography

Use this when the page needs motion tied to reading progression.

## Pinned Story Panel

Use when:

- one preview or product frame should stay fixed
- the surrounding text changes the interpretation of what is shown

Rules:

- the pinned surface must update meaningfully
- each step should communicate one idea
- mobile should flatten or simplify the pinned behavior

## Sticky Stack

Use when:

- cards or modules should accumulate as the user moves down the page

Rules:

- keep the stack count low
- preserve readability of the top card
- avoid excessive overlap on small screens

## Scroll-Led Product Demo

Use when:

- a product workflow needs to be explained with progressive focus

Prefer:

- state highlighting
- subtle internal frame changes
- synchronized copy and product emphasis

Avoid:

- faking a video player with too many unnecessary transitions

## Horizontal Rail

Use when:

- proof, logos, or modules need to move laterally within a vertical story

Rules:

- keep the motion smooth and readable
- use only when horizontal grouping improves comprehension
- never trap the user in a confusing scroll hijack unless the experience absolutely depends on it

## Sequence Design

For any scroll choreography:

1. Define the number of steps
2. Decide what changes at each step
3. Keep one stable anchor in view
4. Limit moving parts
5. Simplify below `768px`
