# Component Principles

Components must reinforce the chosen layout.

## Hero

Every hero needs:

- one dominant message
- one supporting proof or product surface
- one obvious action cluster

Avoid:

- centering everything by default
- stacking too many badges before the headline
- treating screenshots as decoration

## Section Rhythm

Each section should do one primary job:

- explain
- prove
- demonstrate
- convert

Alternate between wider and tighter sections so the page has pacing.

## Cards

Use cards when modularity helps comprehension.

Rules:

- vary scale when the layout benefits from it
- do not repeat identical cards for every section
- keep internal padding generous
- use card backgrounds to separate functions, not to add noise

## CTA Design

Conversion should be visible without dominating every section.

Prefer:

- one primary CTA and one secondary CTA in the hero
- lighter CTA moments later in the page
- consistent button language

Avoid:

- a different CTA treatment in every section
- too many bright accent colors competing with the CTA

## Typography

Type should create hierarchy before decoration does.

Use:

- high contrast between heading and body roles
- consistent width and rhythm within a section
- expressive display choices only when they support the brand

Avoid:

- too many font families
- uniform sizing across sections
- overusing all caps or tiny labels

## Motion

Motion exists to clarify sequence and depth.

Prefer:

- reveal-on-entry
- subtle hover translation
- layered hero depth

Avoid:

- looping motion on too many elements
- long decorative animations
- animating width, height, top, or left

## Responsive Behavior

Mobile layout is a redesign, not a screenshot shrink.

Below `768px`:

- collapse to one strong reading column
- remove overlap that hurts clarity
- preserve the main focal order
- keep padding generous
- reduce secondary decoration first

## Anti-Patterns

Reject outputs that look like:

- centered startup template
- feature-card conveyor belt
- gradient wallpaper with weak structure
- random "premium" shadows without compositional logic
- dense desktop layout simply stacked on mobile
