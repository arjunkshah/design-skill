introducing the system

a smaller, stricter system for layout decisions

The skill reads the brief, selects a layout archetype, chooses variation axes deliberately, and checks the page against a quality rubric before the model stops. That keeps the outputs flexible without making them random.

design to code and back

one shared layout language, not one repeated look

Layout Director does not freeze the model into a single house style. It sets a floor for composition quality, then gives the model controlled degrees of freedom so each brief can produce a different result with the same design bar.

  • different archetypes for different products
  • variation matrix to avoid repetitive outputs
  • component rules that serve the page structure
  • review rubric to catch weak hierarchy before shipping

component range

the page can stay quiet and still show more than one pattern

It needs a vocabulary of stronger UI blocks: bentos, proof strips, framed product areas, editorial rails, sticky scrollytelling, and mixed-density sections that can carry different products.

bento stage

mixed scale cards with one dominant thesis

use bento layouts when the product has multiple capabilities but still needs hierarchy, contrast, and one clear anchor.

proof rail metrics, logos, testimonials, snapshots
product frame hero screenshot with supporting orbit cards
editorial split large copy block paired with modular interactive support
gallery runway staggered showcases for visual brands
scroll choreography sticky demo
01 pin one visual anchor while the narrative moves

sticky sections work when the preview stays legible and the right column advances the explanation.

02 change the surface, not the page shell

the content inside the frame should evolve with the story instead of rebuilding the whole viewport on every step.

03 pair sticky media with modular notes

this is where agents can show flow diagrams, product frames, metrics, or side-by-side comparisons without visual drift.

04 collapse cleanly on mobile

pinned desktop sections should become a normal vertical story below 768px rather than trying to preserve the sticky trick.

01

sticky scrollytelling for explanation-heavy sections

Use this pattern when the page needs to explain a workflow, compare states, or walk through a product sequence while keeping one demo surface in view.

02

swap content states inside the pinned frame

The model should update cards, panels, or diagrams inside the sticky area as the user scrolls, not just repeat text blocks next to a static image.

03

combine with bento and proof modules

Sticky sequences get stronger when they hand off into dense modular sections instead of using the same section rhythm for the whole page.

04

treat it as one pattern in a larger system

The page should mix calm editorial sections, bentos, rails, and one or two richer scroll behaviors rather than turning every block into a gimmick.

anti-slop

hand off boilerplate work. keep the judgment in the loop.

Let the agent handle structure selection, responsive cleanup, and section pacing. Keep the aesthetic and product judgment where it belongs: with the person directing the work.

what it bans

default centered startup templates

no endless feature-card conveyor belts, no random gradients, and no equally weighted sections pretending to be hierarchy.

what it prefers

clear focal points and measured pacing

layouts should widen, compress, pause, and convert with intent instead of repeating one grid forever.

what it protects

mobile that feels redesigned

below 768px, overlap disappears, hierarchy survives, and the page still feels purposeful rather than collapsed.

open source install

install it like a normal skill repo

The repo is meant to install directly as a skill source, then expose `layout-director` as the skill you invoke in your agent.

install npx skills add https://github.com/arjunkshah/layout-skill
invoke $layout-director
1. add the repo as a skill source
2. invoke `layout-director` in a frontend task
3. fall back to `skills/layout-director/` manually if your tool needs it
